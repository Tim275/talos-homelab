# Disaster Recovery Drill Log

Quartal-Drills um Velero+CNPG Restore-Pipeline zu validieren.
Nie verlassen auf "Backup completed" — nur **Restore-Test** beweist Wiederherstellbarkeit.

## Drill-Template

```yaml
apiVersion: velero.io/v1
kind: Restore
metadata:
  name: drdrill-<target>-<YYYYMMDD>
  namespace: velero
spec:
  backupName: <pick-completed-backup>
  namespaceMapping:
    <source-ns>: drdrill-<target>
  includedNamespaces: [<source-ns>]
  restorePVs: false                      # Metadata-restore (no PV-snapshot)
  excludedResources:
    - sealedsecret.bitnami.com           # vermeidet KC-OIDC-conflict
    - replicaset                          # auto-generated by Deployment
    - pod                                 # auto-generated by Controller
    - statefulset.apps                   # would clash with live KC
    - deployment.apps
    - keycloak.k8s.keycloak.org          # operator-managed CR
    - cluster.postgresql.cnpg.io         # CNPG operator-managed
```

## RTO Measurement Methodology

```
RTO = Recovery Time Objective = wie schnell nach Disaster wieder produktiv

Drill RTO 30s misst: K8s-state-restore (ConfigMaps + Secrets + Services + ...)
NICHT enthalten:
  - PVC-Restore (mit restorePVs:true + Volume-Snapshots → 5-20min)
  - CNPG-Database-Replay (WAL-Recovery via Barman → 10-60min für 10GB)
  - Pod-Cold-Start nach restore (image-pull etc → 1-5min)

Voll-Disaster-RTO realistisch: 20-90min für komplette tenant-restore.
```

## Wenn ein RICHTIGER Disaster passiert

```
1. PRE-WORK (immer parat):
   ✓ /tmp/break-glass-prod.yaml (X.509 kubeconfig, im Tresor)
   ✓ /tmp/talosconfig-prod (im Tresor)
   ✓ Diese Datei mit Backup-Liste

2. SCHRITTE:
   a) X.509-kubeconfig laden (OIDC ist down)
   b) Restore-CR template aus dieser Datei kopieren
   c) Backup-Name aus `kubectl get backups -n velero` (oder offline-liste)
   d) Apply → warten
   e) Pod-restart manuell triggern (statefulsets/deployments)
   f) Verify per `kubectl get all -n <target>`

3. POST-DRILL:
   - Diese Datei aktualisieren mit neuem Drill-Eintrag
   - Pre/Post-Snapshots wenn nötig
```
